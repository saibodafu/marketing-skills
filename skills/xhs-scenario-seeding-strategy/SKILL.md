# 技能名称: 小红书场景种草策略

description: 帮助用户判断一个产品在小红书上应该优先种草哪个使用场景、从哪里起势最合理。当用户需要做全量使用场景拆解、四象限场景归类、优先种草场景判断和小红书种草选题时，使用本技能。本技能适合新品种草、产品卖点内容化、品牌场景策略和小红书内容规划场景。

version: 0.1.0

## 使用场景/触发条件

本技能适用于：

- 产品准备在小红书做种草，但还不确定应该优先打哪个使用场景。
- 品牌已有产品卖点，需要把卖点转化为用户真实场景和内容选题。
- 新品上市前，需要判断从主流刚需场景切入，还是从高压强引线场景起势。
- 内容团队需要围绕一个产品建立小红书场景矩阵和选题方向。
- 策略团队需要比较主线、支线、引线、极限场景的品牌价值和优先级。

不适用于：

- 用户只需要单条小红书标题或一篇完整种草笔记，不需要场景策略判断。
- 用户需要基于真实平台数据做达人、笔记、关键词或竞品投放复盘，但没有提供数据来源。
- 用户要求输出具体市场规模、份额、销量、竞品表现等数据，却不允许联网或不提供资料。
- 用户需要抖音投放脚本、千川短视频脚本或电商详情页文案，这些应使用对应技能。

## 触发条件

当用户提出以下需求时，使用本技能：

- “帮我做小红书场景种草策略”
- “这个产品应该优先种草哪个场景”
- “拆一下这个产品的小红书使用场景”
- “全量场景拆解 + 四象限归类”
- “主线场景、支线场景、引线场景、极限场景”
- “帮我判断小红书从哪个场景起势”
- “基于产品卖点输出小红书种草选题”

也可以通过以下方式调用：

```text
/marketing xhs-scenario-seeding-strategy 产品名称：... 产品品类：... 核心卖点：... 目标用户：... 竞品信息：...
```

或：

```text
/marketing 小红书场景种草策略 产品名称：... 产品品类：... 核心卖点：... 目标用户：... 竞品信息：...
```

## 必要输入

用户最好提供以下信息：

| 输入项 | 是否必需 | 说明 |
|---|---|---|
| 产品名称 | 可选 | 有助于输出更贴近品牌和产品表达；缺少时可标注“产品名称未提供” |
| 产品品类 | 必需 | 用于判断真实使用场景、替代方案和用户需求边界 |
| 核心卖点 | 必需 | 用于判断哪些场景最能体现产品差异 |
| 目标用户 | 建议提供 | 用于收窄生活方式、购买动机和小红书内容表达；缺少时可轻量假设 |
| 竞品信息 | 可选 | 可填写竞品品牌、同类产品、旧方案或用户当前替代方案 |

## 缺失信息处理

- 如果缺少产品品类或核心卖点，且无法从上下文合理推断，先追问。
- 如果缺少产品名称，但品类、卖点和目标用户清楚，可以继续分析，并在产品名称处标注“未提供”。
- 如果缺少目标用户，可以基于品类常识做轻量假设，并在输出中标注“假设”。
- 如果缺少竞品信息，不要追问；默认对比常见同类产品、旧方案或替代方案，并标注为假设。
- 所有假设都必须在输出中标注“假设”。
- 不要编造市场规模、份额、销量、预算、调研结论、真实竞品数据、平台热词、案例或用户没有提供的事实。
- 如果用户要求最新趋势、竞品动作、平台热词、公开数据或来源支撑，必须先联网核验，并标注来源和时间。

当必须追问时，只问最关键的问题，不要一次性展开长问卷：

```text
要做小红书场景种草策略，我还需要补充两个关键信息：产品品类是什么？核心卖点是什么？
```

## 工作流程

### 第一步｜理解任务

先判断用户是否已经提供以下产品信息：

- 产品名称
- 产品品类
- 核心卖点
- 目标用户
- 竞品信息

如果关键输入足够，直接进入分析；如果缺少品类或核心卖点且无法推断，先追问。若只缺少非关键输入，做轻量假设并标注。

### 第二步｜分析或处理

以小红书场景策略专家视角，基于用户真实需求、购买动机与生活方式进行分析。

分析时必须坚持：

- 以用户需求为锚点建立细分场景维度。
- 场景要覆盖全面、互不重复。
- 可以先分大场景，再拆细分场景。
- 每个场景都要评估用户市场大小、竞品差异优势和创意出圈潜力。
- “用户市场大小”“竞品优势”“创意出圈度”默认使用定性判断，例如“大 / 中 / 小”“强 / 中 / 弱”，并说明原因。
- 不写空泛营销理论，结论必须回到产品差异、小红书表达和品牌起势机会。

### 第三步｜生成结果

严格按照“输出格式”中的五步结构生成结果，不压缩、不跳步、不改变核心表格字段：

1. 用户输入产品信息
2. 全量使用场景拆解
3. 四象限场景归类
4. 判断最应该先种的场景
5. 小红书种草选题

### 第四步｜检查质量

输出前检查：

- 是否所有场景都基于真实用户需求，而不是产品功能罗列。
- 是否存在高度重复或边界不清的场景。
- 是否每个场景都完成市场大小、竞品优势、创意出圈度判断。
- 是否四象限归类理由清楚。
- 是否只推荐 1 个最应该优先种草的核心场景。
- 是否所有选题都围绕该优先种草场景，没有跳到无关场景。
- 是否选题之间角度有区分度，而不是同一个角度换标题。
- 是否对缺失信息和不确定判断标注了“假设”。

## 第三步｜四象限场景归类

请基于第一步拆解出的全量场景，将它们分别归类到以下四类场景中：

### 1. 主线场景：高频 × 刚需

主线场景代表主流人群、主流价值、主流文化。

判断标准：

- 用户规模大
- 使用频率高
- 需求明确
- 容易形成长期转化
- 适合承接品牌主流生意

### 2. 支线场景：高频 × 非刚需

支线场景用户触达多，但不是强刚需，也不一定是品牌独家优势。

判断标准：

- 使用频率高
- 用户容易理解
- 但需求不够迫切
- 竞品也容易进入
- 差异化相对较弱

### 3. 引线场景：低频 × 刚需

引线场景虽然低频，但在关键时刻不可替代。
判断标准：

- 使用频率不一定高
- 但需求强度很高
- 用户主动寻找解决方案
- 容易成为品牌第一批用户
- 能点燃品牌第一把火
- 具备用户培育和外溢扩散价值
- 容易诱导用户共创、分享和扩散

### 4. 极限场景：低频 × 非刚需

极限场景非常小众而狭窄，但压强高，能够体现产品性能、技术含量或品牌态度。
判断标准：

- 用户规模小
- 使用频率低
- 不一定刚需
- 但画面感强
- 话题性强
- 适合打造传播事件和内容营销

## 质量标准

输出必须满足：

- 严格按照五步结构输出。
- 保留原始提示词中的判断标准、表格字段和选题要求。
- 场景拆解要覆盖全面、互不重复，并且以用户需求为锚点。
- 四象限归类要有明确理由，不能只做标签归类。

- 小红书选题必须围绕优先种草场景展开，不要跳到其他场景。
- 选题必须从用户痛点、场景困扰或真实需求自然过渡到产品卖点。
- 缺少非关键字段时可以做轻量假设，但必须标注“假设”。
- 缺少关键字段且无法推断时先追问。
- 不编造数据、案例、来源或用户没有提供的事实。
