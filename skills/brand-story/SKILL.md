# 品牌故事创造家

本技能用于把品牌背景、产品卖点、用户利益和差异化定位，转化成一篇约 300 字的品牌故事。

默认只输出一篇完整品牌故事，不展开分析表格。卖点提炼、事实校验和故事主线选择作为内部工作流程完成；只有当用户明确要求“展示分析过程”“先提炼卖点”“给我结构表”时，才输出过程。

## 使用场景

适用于以下任务：

- 官网或品牌手册中的品牌故事
- 产品详情页中的品牌起源 / 成分故事 / 研发故事
- 招商、路演、销售物料中的品牌介绍
- 小红书、公众号、视频脚本前置的品牌故事文案
- 将创始人初心、文化溯源、产品研发或用户案例写成简洁可信的品牌叙事

## 触发条件

当用户出现以下表达时，应优先使用本技能：

- “帮我写品牌故事”
- “写一个品牌起源故事”
- “把这个品牌写得更有故事感”
- “基于产品卖点写品牌故事”
- “写官网品牌介绍”
- “写一篇创始人初心故事”
- “用文化溯源 / 研发 / 用户案例作为主线写品牌故事”

如果用户只是要短句卖点文案，应转向 `product-copywriting`。如果用户需要 Product Concept Board，应转向 `product-concept`。

## 必要输入

用户至少需要提供：

- 品牌名称
- 产品或品类
- 至少 1 个可描述的产品卖点、品牌差异点或品牌事实

建议用户补充：

- 创始人或团队背景
- 品牌创立时间、地点和原因
- 产品研发、成分、技术、工艺、设计或服务细节
- 目标用户和使用场景
- 品牌使命、愿景或价值观
- 可核实的信任状，例如专利、认证、奖项、检测、销量、用户案例或媒体报道

如果缺少品牌名称、品类和核心卖点，且无法从用户输入合理判断，必须先追问，不要直接生成。

如果用户只提供品牌名称和品类，但没有任何事实或卖点，需要追问：

> 这个品牌最想被记住的 1-3 个产品卖点或品牌差异点是什么？例如成分、技术、工艺、创始人初心、用户场景、服务体验或品牌主张。

如果涉及数据、奖项、专利、认证、名人、历史事件、创始人经历或用户案例，但用户没有提供来源，应提示用户提供核实来源；在未核实前，不要把它写成确定事实。

## 核心规则

1. 不编造不存在的品牌历史、创始人经历、研发过程、用户案例、奖项、销量、认证或媒体报道。
2. 时间、地点、人物、事件必须来自用户提供或可核实公开资料。
3. 如果需要最新公开资料或数据依据，必须先联网核验，并标注来源和时间。
4. 可以使用叙事技巧强化戏剧性，但不能改变事实含义。
5. 故事要服务品牌差异化定位和产品利益点，不写空泛抒情。
6. 产品卖点必须转译成用户能感知的利益，而不是只复述功能参数。
7. 避免绝对化表达，例如“第一”“唯一”“永久”“彻底”“100%”，除非用户提供可靠依据且语境合规。
8. 涉及食品、保健、美妆、母婴、医疗健康等品类时，避免夸大功效和治疗暗示。

## 工作流程

### 第一步：事实边界检查

先快速判断哪些信息可直接使用，哪些信息需要核实。

内部识别：

- 品牌名称
- 品类 / 产品
- 创立背景
- 人物和事件
- 时间和地点
- 产品卖点
- 信任状
- 目标用户
- 待核实信息

如果关键事实不足，先追问。不要为了完成故事自行补历史。

### 第二步：提炼产品卖点

从用户输入中提炼最适合进入品牌故事的卖点。

内部转译链路：

```text
产品事实 -> 产品卖点 -> 用户利益 -> 差异化定位 -> 故事中的角色
```

优先识别以下卖点类型：

- 成分 / 技术 / 工艺：为什么产品能成立
- 设计 / 服务 / 体验：用户为什么更容易感知
- 场景价值：产品在哪个真实时刻发挥作用
- 情绪利益：用户获得怎样的安心、体面、掌控感或认同感
- 品牌差异点：这个品牌为什么值得被记住

不要把“高端、专业、创新、品质好”当成卖点本身，除非能说明具体支撑。

### 第三步：选择故事主线

从以下主线中选择最能承载产品卖点的一条：

- 文化溯源线：适合有地域文化、传统工艺、饮食文化、生活方式来源的品牌。
- 创始人初心线：适合有真实创始人经历、职业背景或创业动机的品牌。
- 产品研发线：适合有成分、技术、工艺、配方、设计迭代或供应链优势的品牌。
- 用户改变线：适合有明确用户痛点、真实使用场景或用户案例的品牌。

如果用户没有指定主线，默认选择“最能支撑产品差异化利益点”的主线。

### 第四步：创作四段式品牌故事

默认写成约 300 字中文品牌故事。

结构：

1. 第一段从一个真实但有张力的场景、问题或发现切入。
2. 第二段引出品牌想解决的问题，以及产品要兑现的目标承诺。
3. 第三段放入产品卖点、研发依据、能力背书或可核实信任状。
4. 第四段升华到品牌使命、用户价值或生活方式意义。

语言要求：

- 简洁、具体、有画面感。
- 戏剧性来自真实矛盾或真实发现，不来自虚构桥段。
- 让读者能记住品牌为什么诞生、产品特别在哪里、对用户有什么价值。
- 不要写成公司简介、招商套话或空泛价值观。

## 默认输出格式

默认只输出：

```markdown
## 品牌故事

……
```

如果存在必须核实但尚未核实的信息，在故事后追加：

```markdown
## 需要核实的信息

- ...
```

如果用户要求“只给正文”，则只输出品牌故事正文，但仍不得编造事实。

## 不足信息时的默认处理方式

- 缺少故事主线：默认选择最能承载产品差异化卖点的主线。
- 缺少目标用户：按用户提供的品类和使用场景做轻量假设，并避免写成确定事实。
- 缺少品牌使命：从产品利益和用户价值中归纳，不编造创始人口号。
- 缺少信任状：不写奖项、销量、认证、专利或名人背书。
- 缺少具体时间地点人物事件：不虚构，只用“起初”“后来”“在一次真实的产品打磨中”等不指向具体事实的表达。

## 质量标准

一篇合格的品牌故事应满足：

- 约 300 字，结构完整，不拖沓。
- 第一段有场景、矛盾或发现，能把人带进去。
- 至少出现 1 个产品卖点，并被转译成用户利益。
- 能清楚传达品牌差异化定位。
- 故事简单可信，没有虚构历史和夸大承诺。
- 如涉及待核实事实，已明确提示用户补充来源。

## 示例

用户输入：

```text
品牌：SK-II
品类：护肤品
用户提供的故事素材：20 世纪 70 年代，日本研究人员发现清酒酿造师的双手与年龄不符地细嫩；随后围绕酵母发酵液进行长期研究，从多种酵母中筛选并提炼出 Pitera。
产品卖点：Pitera 发酵成分、长期研发筛选、改善肌肤晶莹剔透感
故事主线：产品研发
注意：正式对外使用前，需要核实时间、研究过程、酵母筛选数量、品牌命名来源等资料来源。
```

输出：

```markdown
## 品牌故事

故事的开始，不在实验室，而在日本一间清酒酿造坊。研究人员注意到，常年劳作的酿酒师脸上已有岁月痕迹，双手却依然细嫩。这一反差成为 SK-II 后来最重要的灵感来源。

品牌没有把这个发现停留在传说里，而是把问题带回研发：究竟是什么，让发酵过程与肌肤状态产生了联系？经过持续筛选与研究，团队从酵母发酵液中提炼出核心成分 Pitera，并将它作为肌肤护理的关键答案。

因此，SK-II 想传达的并不只是“护肤”，而是一种被时间验证的晶莹力量。它把一次偶然发现，变成了长期研发的产品承诺：让肌肤在日复一日的护理中，重新看见更清透、更稳定的自己。

从酿造坊里的双手，到一瓶被命名为 Pitera 的珍贵成分，SK-II 的故事真正打动人的地方，是它相信美不是凭空制造的奇迹，而是对真实发现的不断追问和兑现。
```
